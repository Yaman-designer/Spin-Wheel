Custom site script

Validated on 15 Jan 2024 • Last edited on 27 Jun 2025

Overview

Use this guide if your website is built with a custom tech stack (HTML, React, Vue, Laravel, etc.). You will copy a single script tag from Wheelluck and paste it into the <head> of every page where you want the promotion to appear. Once the script loads, your active promotion displays automatically.

Before you start

  • You need an active promotion in Wheelluck.
  • You must have access to edit your website’s HTML templates or theme files.
  • Make sure you can publish or deploy changes after editing the code.
  • If you use a staging environment, install and test there first.

Step 1 — Copy the Wheelluck script

  1. Log in to Wheelluck and open the promotion you want to install.
  2. Go to Install widget.
  3. In the Custom Website tab, click the copy button next to the script tag.
  4. Keep this tab open in case you need to copy the script again.
Tip: The script string is unique per promotion. If you duplicate a promotion, repeat this step to get the new link.

Step 2 — Paste the script into your website

  1. Open your site’s layout file(s) in your code editor or CMS.
  2. Locate the closing </head> tag. The Wheelluck script should appear before this closing tag.
  3. Paste the script exactly as copied. Example:
<script src="https://cdn.wheelluck.com/promotions/abc123.js" type="text/javascript" id="wheelluck-script" defer></script>
  1. Save the file and publish/deploy your changes.
Important: Keep the attribute id="wheelluck-script". Removing it prevents Wheelluck from detecting the installation automatically.

Step 3 — Verify the installation

  1. Return to the Install widget page in Wheelluck.
  2. Enter the page URL in the Retry checker and click the button.
  3. If the script is found, you will see Setup is successful. If not, review the troubleshooting tips below.
  4. Visit your live site in an incognito/private window and trigger the promotion to confirm it loads.

Optional adjustments

  • Single-page applications: Insert the script in the main HTML shell (e.g., public/index.html) so it runs regardless of route changes.
  • Tag managers: You can load the script via Google Tag Manager or similar tools. Ensure it fires on every page view.
  • Lazy loading: The script uses defer. Avoid wrapping it in additional lazy loaders which may delay the popup beyond the page view.

Troubleshooting

  • Checker says “script not found”:
    • Double-check the script tag is present inside the <head> on the published site.
    • Make sure caching/CDN layers have been cleared after deployment.
    • Verify the URL you entered in the checker exactly matches the page where the script lives.
  • Promotion does not appear:
    • Confirm the promotion status is Active in Wheelluck.
    • Check if other scripts or popup blockers on your site are preventing the popup.
    • Try disabling aggressive cookie banners or script blockers temporarily to test.
  • SPA route change issue: If the popup shows only on first load, trigger a manual page reload after route changes or fire window.wheelluckReload() after navigation.
  • Multiple promotions: Only one promotion can be active per script. Switch the active promotion in Wheelluck if the wrong campaign appears.

Need more help?

If you run into edge cases (custom build pipelines, SSR frameworks, etc.), contact Wheelluck support at info@wheelluck.com with details about your stack and deployment process.
